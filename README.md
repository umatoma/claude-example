# claude-example

## Backend Server

Node.js + Express.js を使用した Hello World サーバーです。

### セットアップ

```bash
cd backend
npm install
```

### 起動方法

**本番モード:**
```bash
cd backend
npm start
```

**開発モード（ホットリロード）:**
```bash
cd backend
npm run dev
```

### 動作確認

サーバー起動後、ブラウザまたはcurlでアクセス:

```bash
curl http://localhost:3000
```

レスポンス: `HELLO WORLD`

## ベストプラクティス
- コンテキストがいっぱいになると精度が落ちるため、不必要な情報をコンテキストに含めない
- いきなり実装すると間違った問題解決をしてしまうため、探索→計画→実装のステップを踏む
- 暗黙的な意図は読み取れないため、プロンプトには具体的な情報を含める

## CLAUDE.md
- セッションごとに読み込まれる
- プロジェクト全体に適用される情報のみを含める
- ドメイン知識やプロジェクト内の一部のみに適用される情報はスキルを使用する
- 指示に強調（重要・必須、など）を含めることで指示内容の遵守率を上げられる
- `@path/to/import`で別ファイルを参照できる
- プロジェクト内のサブディレクトリにも`CLAUDE.md`ファイルを設置できる

| ✅ 含める | ❌ 除外 |
| ----- | ----- |
| Claudeが推測できないBashコマンド | Claudeがコードを読んで理解できること |
| デフォルトとは異なるコードスタイルルール | Claudeがすでに知っている標準的な言語慣習 |
| テストの手順と推奨テストランナー | 詳細な API ドキュメント (代わりにドキュメントへのリンク) |
| リポジトリのエチケット（ブランチの命名、PRの規則） | 頻繁に変更される情報 |
| プロジェクト固有のアーキテクチャ上の決定 | 長い説明やチュートリアル |
| 開発環境のルール（必要な環境変数） | コードベースのファイルごとの説明 |
| よくある落とし穴やわかりにくい動作 | 「きれいなコードを書く」といった自明の習慣 |


## スキル
- プロジェクト・チーム・ドメインに固有な情報でClaudeの知識を拡張する
